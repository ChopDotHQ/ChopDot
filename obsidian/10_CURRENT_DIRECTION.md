# Current Direction

## Active now

- ChopDot is a **shared commitment** product.
- The current MVP is **coordination-first**, not custody-first.
- The first implementation proof is the **chapter / closeout loop**.
- The strongest wedge is still **group deposits**, even though the simplest current technical recovery path is **expense closeout**.

## Supporting current work

- API-ready architecture
- founder validation before builder exposure
- modular auth / DB / proof boundaries
- subscription-led monetization

## Deferred

- builder platform as a primary product
- public API productization
- tokenization as a first-class product surface
- agent execution
- custody / pooled funds / yield

## Rejected for now

- “just another expense app”
- booking-engine-first
- token-first economics
- hidden spread on user money
