## FACTS

- Ecosystem-facing apps are funded through multiple mechanisms, not one:
  - treasury grants
  - retroactive public goods funding
  - ecosystem grant programs
  - direct protocol/foundation grants
  - referrals / revenue share
  - embedded fee capture
  - validator / infra revenue
- Current official examples:
  - **Nova Wallet**:
    - explicitly documents treasury grants from Kusama, Polkadot, Moonbeam, and KILT. [Nova treasury bounties](https://docs.novawallet.io/nova-wallet-wiki/misc/treasury-bounties)
    - in its 7th Polkadot proposal it claims:
      - `100,000+` monthly active users
      - `13.4M DOT` staked via Nova by `12.3k` users
      - about `18%` of all DOT stakers staking via Nova
      - about `71.5%` share of ecosystem wallet referral-code users for swaps
      - it also describes revenue sources beyond treasury: validators, Hydration referral fees, and ecosystem-specific features. [Nova 7th proposal](https://api.polkassembly.io/referenda/1702)
  - **Web3 Foundation Decentralized Futures**:
    - explicitly funds teams that contribute to ecosystem growth and can “scale to self-sufficiency.”
    - program supports grants, investments, and service agreements. [Decentralized Futures](https://futures.web3.foundation/)
  - **Optimism Retro Funding**:
    - funds projects that increased demand for blockspace and drove value to the Collective.
    - required measurable criteria such as unique addresses, activity days, public code, verified contracts, and KYC. [Retro Funding 4 guidelines](https://gov.optimism.io/t/retro-funding-4-onchain-builder-application-guidelines/8165)
  - **Solana grant opportunities**:
    - explicitly fund user-facing staking UIs that improve delegation, validator distribution, and stake management. [Solana multi-validator staking UI grant](https://solana.org/grants/opportunity-multi-validator-staking-ui.pdf) [Solana automated stake delegation UI grant](https://solana.org/grants/opportunity-automated-stake-delegation-and-rebalancing-ui.pdf)
  - **MetaMask**:
    - captures direct user fees on swaps via a `0.875%` MetaMask fee. [MetaMask swaps fee](https://support.metamask.io/manage-crypto/move-crypto/swap/user-guide-swaps)

## INFERENCES

- Systems like Nova win funding because they can show:
  - user adoption
  - chain/protocol usage driven through their product
  - measurable ecosystem value
  - multiple paths to self-sustainability
- Ecosystems will pay when an app can prove:
  - it brings users
  - it retains them
  - it drives valuable onchain or protocol activity
  - it does so in a way that is visible and attributable
- ChopDot can pursue a similar model later, but only if it is designed for:
  - attribution
  - neutrality
  - measurable impact

## ASSUMPTIONS

- ChopDot wants to remain coordination-first while later becoming valuable enough to chains/protocols that they fund, incentivize, or partner with it.
- ChopDot wants fair, legible value capture rather than opaque extraction.

## 1. The Main Funding Models Used By Systems Like This

## A. Treasury / grant funding

How it works:
- foundations, DAOs, or treasuries fund products that help ecosystem growth

Why they pay:
- the app improves user onboarding, staking, governance, or protocol usage
- the app fills a strategic ecosystem gap

Examples:
- Nova Wallet treasury grants
- Web3 Foundation Decentralized Futures
- Solana ecosystem grant opportunities

## B. Retroactive funding

How it works:
- the ecosystem rewards projects after measurable impact is already visible

Why they pay:
- the project demonstrably increased demand, users, or usage

Example:
- Optimism Retro Funding

## C. Revenue-sharing / referrals / integrations

How it works:
- the app drives transaction flow or protocol usage and gets a share

Why they pay:
- the app is an attributed distribution layer

Examples:
- Nova Wallet + Hydration referral system
- validator revenue for Nova’s team-run validators

## D. Direct user fees

How it works:
- the app charges users for integrated actions

Why it works:
- the app controls the UX and extracts a margin directly

Example:
- MetaMask swap fee `0.875%`

## 2. How They Are Measured

Across these systems, the important metrics are usually:

- monthly active users
- active wallets / unique addresses
- total value staked
- staking participation share
- swap/bridge/trading volume
- referral-code share
- unique users performing meaningful actions
- days active / retention
- number of ecosystems or chains supported
- number of public integrations
- open-source and verifiable code
- security and compliance posture

## 3. What This Means For ChopDot

If ChopDot wants ecosystems to eventually pay it to keep driving activity, it needs to become measurable as an ecosystem value layer.

That means ChopDot should be able to show:

- how many commitments were created
- how many users were onboarded to a specific chain or protocol flow
- how many commitments funded a protocol-specific action
- how much value / volume / stake / participation was routed through ChopDot
- how many users returned
- what share of a protocol’s user activity ChopDot influenced

## Today

Primary business:
- direct subscription software

Why:
- ChopDot needs a real business now

## Tomorrow

Additional ecosystem-aligned revenue:
- protocol / chain grants

- revenue share from attributed actions
- partner/service agreements

Why:

## 5. What ChopDot Should Measure From Day One

If this future matters, ChopDot should start designing metrics now.

## Core product metrics

- commitments created
- commitments completed
- repeat organizer rate
- active users by commitment type

## Ecosystem attribution metrics

- commitments using protocol-specific rails
- users onboarded into chain/protocol actions
- protocol-specific transaction count initiated through ChopDot
- protocol-specific volume or value coordinated through ChopDot
- retained users by ecosystem

## Partner-value metrics

- conversion rate from “invited” to “activated”
- time to funded state
- time to closeout
- number of successful release/closeout completions

## 6. What Would Make ChopDot Payable By Ecosystems

For a chain, treasury, or protocol to justify paying ChopDot, these things should be true:

1. ChopDot brings users or activity they care about
2. the contribution is attributable
3. the product is neutral enough to be trusted
4. the metrics are legible
5. the product is reliable and open enough to be scrutinized

## 7. What Would Make ChopDot Unpayable

- no measurable attribution
- vague ecosystem claims
- no proof of repeat usage
- no proof that activity actually reaches the protocol
- too much chain bias too early
- weak trust/security posture

## 8. ChopDot Strategy Implication

ChopDot should not depend on ecosystem funding as the first business model.

But it should absolutely be designed so that later it can credibly say:

**we bring users, commitments, volume, and retained activity into your ecosystem in a measurable way**

That is the bridge from:
- subscription product
to
- ecosystem-relevant layer

## 9. Recommendation

Design ChopDot around three monetization layers:

1. direct subscription now
2. protocol-attributed ecosystem funding and partner agreements later
3. premium builder/API and trust-action pricing later

And design measurement so ecosystems can eventually answer:

- how much usage did ChopDot create?
- how sticky was it?
- was it worth funding?

## Best One-Line Summary

**Systems like Nova get funded because they can prove they drive measurable ecosystem value; if ChopDot wants the same outcome, it must be designed from the start to create, attribute, and report that value clearly.**
