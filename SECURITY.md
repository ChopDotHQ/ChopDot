# Security Policy

If you believe you have found a security issue in ChopDot, please report it privately to [security@chopdot.app](mailto:security@chopdot.app).

## Reporting Guidelines

- Do not open a public GitHub issue for suspected vulnerabilities.
- Include clear reproduction steps, affected routes or files, and any screenshots or logs that help explain the issue.
- If the issue involves secrets, credentials, or user data exposure, say so explicitly in the report.

## Scope

This policy covers the actively maintained `main` branch and the production deployment that follows it.

## What To Expect

- We will review reports privately.
- We may ask follow-up questions or request a minimal reproduction.
- We will decide on remediation, disclosure timing, and release handling after review.

## Notes

- Do not include real private keys, passwords, or live secrets in your report.
- If you accidentally exposed credentials in a fork or branch, rotate them first and mention that in the report.
